🚀 INSTRUCCIONES RÁPIDAS DE INICIO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣ Restaurar dependencias:
   dotnet restore

2️⃣ Compilar (ya completado ✅):
   dotnet build

3️⃣ Crear migraciones:
   cd ConferenSpace.API
   dotnet ef migrations add InitialCreate --project ../ConferenSpace.Infrastructure

4️⃣ Aplicar migraciones:
   dotnet ef database update --project ../ConferenSpace.Infrastructure

5️⃣ Ejecutar la API:
   dotnet run

6️⃣ Acceder a Swagger:
   https://localhost:7123/swagger

🆘 SOLUCIÓN DE PROBLEMAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problema: "Connection refused"
Solución: Verifique que SQL Server LocalDB esté corriendo
         sqllocaldb start MSSQLLocalDB

Problema: "Migrations history table could not be created"
Solución: Borre la base de datos: dotnet ef database drop
         Recrele: dotnet ef database update

Problema: "Cannot restore packages"
Solución: dotnet nuget locals all --clear
         dotnet restore

Problema: "The API doesn't start"
Solución: Verifique puerto 7123, logs, y appsettings.json

Para más ayuda, revise MIGRACIONES.md
